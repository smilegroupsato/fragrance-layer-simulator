# Project Rules / プロジェクトルール

- This is Fragrance Layer Simulator v0.1.
- これは Fragrance Layer Simulator v0.1 です。

- Keep the scope small.
- スコープは小さく保ってください。

- Do not add authentication.
- 認証を追加しないでください。

- Do not add a database.
- データベースを追加しないでください。

- Do not add external APIs.
- 外部 API を追加しないでください。

- Do not add scraping.
- スクレイピングを追加しないでください。

- Do not add image upload.
- 画像アップロードを追加しないでください。

- Do not invent official perfume notes.
- 公式香水ノートを推測・捏造しないでください。

- Keep official information, AI-style layer analysis, public discourse, and user notes separated.
- 公式情報、AI風レイヤー分析、一般的な言説、自分のメモは必ず分離してください。

- Keep internal IDs, routes, and TypeScript field names in English.
- 内部 ID、route、TypeScript field 名は英語のまま維持してください。

- Visible UI may be Japanese.
- 表示 UI は日本語で構いません。

- Glossary item names should be Japanese-primary with original names as secondary display.
- 用語集項目名は日本語を主表示、原語・英語名を副表示にしてください。

- Reference perfume product and brand names should be original-primary with Japanese/kana as secondary display.
- 参照香水の商品名・ブランド名は原語を主表示、日本語・カナ表記を副表示にしてください。

- Run `pnpm run build` before reporting completion.
- 完了報告前に `pnpm run build` を実行してください。

- Use static TypeScript data first.
- まず静的 TypeScript データを使ってください。

- Keep the app browser-based and simple for v0.1.
- v0.1 ではブラウザベースでシンプルなアプリとして保ってください。
